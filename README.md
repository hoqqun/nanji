# üïê nanji

**nanji** (meaning ‚Äúwhat time is it?‚Äù in Japanese) is a lightweight Rust-based CLI tool  
that displays the current time across multiple world cities such as Tokyo, Dallas, and London.

It‚Äôs simple, fast, and perfect for anyone who frequently checks time differences between Japan üáØüáµ and the U.S. üá∫üá∏.

## üöÄ Features

- Displays current time for multiple cities (Tokyo, Dallas, New York, London, Delhi)
- `--zones` option to show only selected time zones
- `--zones` option to show only selected time zones
- `-a, --alias` to label output using your alias names (instead of canonical IANA names)
- Colorful, easy-to-read terminal output
- Time automatically converted from UTC to local zones

## ‚öôÔ∏è Configuration (optional)

You can pin the display to only your favorite timezones by creating a config file.

Config file location (per OS):

- Linux (XDG): `$XDG_CONFIG_HOME/nanji/config.toml` (or `~/.config/nanji/config.toml`)
- macOS: `~/Library/Application Support/nanji/config.toml`
- Windows: `%APPDATA%\nanji\config.toml` (typically `C:\\Users\\<You>\\AppData\\Roaming\\nanji\\config.toml`)

Notes:
- The path is resolved using the OS defaults (same as Rust's `dirs::config_dir()`).
- Create the `nanji` directory if it doesn't exist.

Example `config.toml`:

```toml
# IANA time zones
zones = [
	"Asia/Tokyo",
	"America/Chicago",      # Dallas (CST/CDT)
	"America/Los_Angeles",  # California (PST/PDT)
	"America/New_York",
]

# Optional: aliases (your preferred short names)
[aliases]
# left = how you type, right = canonical IANA time zone
tokyo = "Asia/Tokyo"
dallas = "America/Chicago"
california = "America/Los_Angeles"
ny = "America/New_York"
```

Runtime priority:

1. `--zones` CLI option (comma-separated) e.g. `--zones Asia/Tokyo,America/Chicago`
2. Config file `zones`
3. Fallback to show all zones

Aliases notes:

- Aliases are case-insensitive for lookup (`Tokyo`, `TOKYO`, `tokyo` all match).
- Built-in aliases exist (tokyo/dallas/california/newyork/ny etc.) but your config overrides them if keys collide.

### Examples

- Show only two zones by canonical names but label them with aliases:

```bash
nanji --zones Asia/Tokyo,America/Chicago --alias
```

- Base time from Tokyo 09:00, two outputs labeled with aliases:

```bash
nanji --tokyo 09:00 --zones Asia/Tokyo,America/New_York -a
```
